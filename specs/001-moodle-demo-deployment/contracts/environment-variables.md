# Contract: Environment Variables

**Feature**: `001-moodle-demo-deployment`  
**Interface**: Two `.env` file locations on the DigitalOcean Droplet (never committed to Git)  
- `.env` — shared infrastructure variables (Traefik, MariaDB, Redis)  
- `clients/{name}/.env` — per-client variables (one file per provisioned client)  
**Templates**: `.env.example` and `clients/_template/.env.example` (committed — documents all variables, no real values)

---

## Part A: Shared Infrastructure Variables (`.env`)

These variables configure the shared Traefik, MariaDB, and Redis services.
They are required on the Droplet before running `docker compose up`.

### Networking

| Variable | Example | Description |
|----------|---------|-------------|
| `ACME_EMAIL` | `admin@example.com` | Email for Let's Encrypt certificate registration. Receives expiry warnings. Applies to all client domain certificates. |

### Database (MariaDB)

| Variable | Example | Description |
|----------|---------|-------------|
| `MARIADB_ROOT_PASSWORD` | `SecureR00t!` | MariaDB root user password. Used by the health check probe and by `scripts/add-client.sh` to create per-client databases. Not used by Moodle directly. |
| `MARIADB_PASSWORD` | `SecureDB!` | Password for the shared `moodle` database user. This user is granted access to each client database as it is provisioned. Must be consistent with all `MOODLE_DATABASE_PASSWORD` values in client `.env` files. |

> Note: `MARIADB_DATABASE`, `MARIADB_USER`, and `MARIADB_CHARACTER_SET` are hard-coded in `docker-compose.yml` and do not need to be in `.env`.

## Part B: Per-Client Variables (`clients/{name}/.env`)

Each client has its own `.env` file generated by `scripts/add-client.sh` from the `clients/_template/.env.example` template. These are **REQUIRED** for each client. All credential values are auto-generated by `add-client.sh`; this table documents what each variable controls.

### Client Identity

| Variable | Example | Description |
|----------|---------|-------------|
| `CLIENT_NAME` | `acme-corp` | Client slug — lowercase alphanumeric and hyphens only. Used in service names (`moodle-acme-corp`), volume names, and Redis key prefix. Must be unique across all clients. |
| `CLIENT_DOMAIN` | `acme-corp.com` | Client's public domain. Must have a DNS A record pointing to the Droplet's IP before provisioning. Used as Moodle's `wwwroot` and as the Traefik `Host()` routing rule. |

### Moodle Application

| Variable | Example | Description |
|----------|---------|-------------|
| `MOODLE_USERNAME` | `admin` | Client site admin username. Set on first container start; cannot be changed by re-setting this variable after first run. |
| `MOODLE_PASSWORD` | *(auto-generated)* | Client site admin password. Auto-generated as 32-char hex string. Minimum 8 chars, mixed case + digit. |
| `MOODLE_EMAIL` | `admin@acme-corp.com` | Client admin email address. |
| `MOODLE_SITE_NAME` | `Acme Corp Learning` | Site name shown in the browser title and Moodle header. Defaults to `{CLIENT_NAME} Learning` unless overridden. |
| `MOODLE_HOST` | `acme-corp.com` | Must match `CLIENT_DOMAIN`. Used by Moodle as `wwwroot`. |

### Database

| Variable | Example | Description |
|----------|---------|-------------|
| `MOODLE_DATABASE_HOST` | `mariadb` | Always `mariadb` — the shared MariaDB container service name. |
| `MOODLE_DATABASE_PORT_NUMBER` | `3306` | Always `3306`. |
| `MOODLE_DATABASE_USER` | `moodle` | The shared MariaDB user. Matches `MARIADB_USER` in the shared `.env`. |
| `MOODLE_DATABASE_PASSWORD` | *(from shared `.env`)* | Must match `MARIADB_PASSWORD` in the shared `.env`. |
| `MOODLE_DATABASE_NAME` | `moodle_acme_corp` | Per-client database name. Auto-set by `add-client.sh` as `moodle_{CLIENT_NAME_underscored}`. |

### Redis Session

| Variable | Example | Description |
|----------|---------|-------------|
| `MOODLE_EXTRA_CONFIGPHP` | *(multi-line)* | PHP config injected into Moodle’s `config.php`. Configures Redis session handler with a per-client session key prefix to ensure session isolation. Set by `add-client.sh`. Example: `$CFG->session_handler_class='\core\session\redis'; $CFG->session_redis_host='redis'; $CFG->session_redis_prefix='moodle_acme_corp_session_';` |

## Optional Infrastructure Variables

These variables have working defaults and are set in the shared `.env` file.

| Variable | Default | Description |
|----------|---------|-------------|
| `REDIS_AOF_ENABLED` | `no` | Enable Redis Append-Only-File persistence. `yes` increases durability but adds I/O overhead. |

## Variable Validation Rules

The `deploy.yml` workflow validates shared variables before running `docker compose up`:

```bash
# Checked in GitHub Actions pre-deploy step (shared infra)
required_vars=(ACME_EMAIL MARIADB_ROOT_PASSWORD MARIADB_PASSWORD)
for var in "${required_vars[@]}"; do
  [[ -z "${!var}" ]] && echo "ERROR: $var is not set" && exit 1
done
```

The `scripts/add-client.sh` validates per-client variables before provisioning:

```bash
# Checked in add-client.sh
required_client_vars=(CLIENT_NAME CLIENT_DOMAIN MOODLE_USERNAME MOODLE_PASSWORD
                      MOODLE_EMAIL MOODLE_SITE_NAME MOODLE_DATABASE_NAME)
for var in "${required_client_vars[@]}"; do
  [[ -z "${!var}" ]] && echo "ERROR: $var is not set" && exit 1
done
```

## Security Constraints

- The `.env` file MUST NOT be committed to the repository.
- `.gitignore` MUST contain an entry for `.env`.
- `.env.example` MUST be committed with placeholder values (e.g., `MOODLE_PASSWORD=CHANGE_ME`).
- All passwords MUST be randomly generated (minimum 16 characters, mixed case + digits + symbols).
- The `MARIADB_ROOT_PASSWORD` and `MARIADB_PASSWORD` MUST be different values.
- Redis runs without a password on the isolated internal network only; the Redis port (6379) MUST NOT be published to the host (`0.0.0.0`).
